Command Modules Minimum Structure

Every command is an async function with passed args: message, args, and client.

Passed kwargs

Name Description
stats .01ms timestamps representing process times of the bot
cmds The command modules
dlibs The dynamiclib modules
bot_start The unix timestamp of when the kernel started
main_version The version string of the kernel
kernel_ramfs Ramfs passed from kernel
ramfs Ramfs passed from kernel

File format

category_info

Type: hmap, required fields:

Name Description
name Internal name of module
pretty_name Prettified name of module used in printing to end user
description A short description of the module

commands

Type: hmap, for each command create an hmap within with the name of the command with the following required fields:

Name Description
pretty_name A prettified name of the command used for printing to user
description A short description of what the command does
permission The required permission level of the command, can be everyone, moderator, administrator, or owner
cache Whether to keep, regenerate, or purge cache on command completion
execute A map to the function this command runs

version_info

Type: string; a string representing the version of the command module
